# Payment Card Industry Data Security Standard (PCI DSS) v4.0.1

## What it is
Global standard for organizations that store, process, or transmit cardholder data. Version 4.0.1 introduces customized approach for validation, enhanced authentication requirements, and targeted risk analysis methodology.

## Who it applies to
Any entity that stores, processes, or transmits payment card data — merchants, payment processors, acquirers, issuers, and service providers.

## Key obligations
- Install and maintain network security controls (firewalls, network segmentation)
- Apply secure configurations to all system components
- Protect stored account data using encryption and masking
- Encrypt transmission of cardholder data across open, public networks
- Protect systems against malware and regularly update antivirus software
- Develop and maintain secure systems and software
- Restrict access to cardholder data by business need-to-know
- Identify users and authenticate access to system components
- Restrict physical access to cardholder data
- Log and monitor all access to network resources and cardholder data

## Maximum penalty
Card brand fines: $5,000–$100,000/month; acquiring bank fines; increased transaction fees; loss of card processing capability

## Enforcement deadline
v4.0.1 effective immediately; future-dated requirements mandatory by March 31, 2025
